## Feature implementation order

### Task 1: Binder and scene management polish

Goal:
Make the binder useful for actual manuscript work.

Implement:

1. Rename scene from UI.
2. Create scene with generated id and editable title.
3. Delete scene with confirmation.
4. Duplicate scene.
5. Move scene up/down in binder order.
6. Persist binder order in manifest or binder storage.
7. Keep active document selection stable after create/rename/delete/move.

Acceptance:

- Tests cover create, rename, duplicate, delete, and reorder persistence.
- UI status messages identify the affected scene.
- Saving after rename updates visible binder title and document metadata.
- Large manuscript path remains usable after binder refresh.

---

### Task 2: Scene metadata panel / inspector MVP

Goal:
Add the first real writing-management surface beside the editor.

Implement a simple inspector for the active scene:

- Synopsis
- Status: draft / revised / final / excluded
- Tags
- Target character count
- Current character count display
- Updated timestamp

Storage:

- Use existing metadata model if available.
- If missing, add a minimal side metadata structure or extend manifest carefully.
- Keep manuscript body separate from management metadata where practical.

Acceptance:

- Metadata saves and reloads.
- Changing synopsis/status/tags does not rewrite the editor body unnecessarily.
- Tests cover metadata roundtrip.
- UI labels are Korean.

---

### Task 3: Search UI improvement

Goal:
Turn search from a raw list into a usable manuscript navigation tool.

Implement:

1. Search result includes scene title, snippet, and rough location if available.
2. Selecting a result loads the scene.
3. Highlight or place caret near the matched text if practical.
4. Empty query and no-result states show clear Korean messages.
5. Search should use SQLite/index path when available.

Acceptance:

- Tests cover search result formatting and selection behavior where practical.
- Search does not run on every keystroke.
- Large document search remains user-triggered.

---

### Task 4: Snapshot / backup MVP

Goal:
Protect real manuscript work before more editing features land.

Implement:

1. Manual snapshot command for active scene.
2. Snapshot file written under snapshots/scene-id/.
3. Snapshot list for active scene.
4. Restore snapshot with confirmation.
5. Optional snapshot before destructive actions: delete, restore, large overwrite.

Acceptance:

- Snapshot file is human-readable or clearly derived from current document storage.
- Tests cover snapshot create/list/restore.
- Restore updates editor, binder title if needed, and save state.

---

### Task 5: Export MVP

Goal:
Make the workbench produce usable manuscript output.

Implement:

1. Export current scene to TXT.
2. Export included scenes in binder order to one TXT file.
3. Export destination under exports/.
4. Status message shows output path.
5. Excluded scenes are skipped in full manuscript export.

Acceptance:

- Tests cover binder-order export.
- Tests cover excluded scene skip.
- Output is UTF-8 and Korean text survives.
- Export is explicit/user-triggered.

---

### Task 6: Command palette MVP

Goal:
Support the customization spine with an actual command surface.

Implement:

1. Command palette window/dialog.
2. Search commands by name/id/category.
3. Execute selected command.
4. Default shortcut, for example Ctrl+Shift+P.
5. Include project, document, search, preview, focus, help, snapshot, export commands.

Acceptance:

- Command registry remains the source of truth.
- Command palette lists registered commands, not hardcoded duplicate buttons.
- Tests cover command filtering and execution routing where practical.

---

### Task 7: Toolbar/command slot editing MVP

Goal:
Make custom cockpit idea visible.

Implement minimal command slot customization:

1. Show current command slots.
2. Allow replacing a slot with another registered command.
3. Persist command slot layout.
4. Restore layout on app startup.
5. Provide reset-to-default.

Acceptance:

- Tests cover save/load/reset of command slots.
- UI labels are Korean.

### Task 8: Focus session options UI

Goal:
Make focus mode user-configurable.

Implement:

1. Focus duration setting.
2. Early-exit confirmation length setting.

## Performance guardrails during feature work

Use these as regression checks, not as the main project track.

Baseline to preserve:

```text
85,000 paragraphs
about 10,000,000 characters including spaces
paste body into editor: about 2 seconds
save: under 500 ms
```

Feature work must keep these rules:

- Editor text change path handles only cheap state changes.
- Search, preview, export, indexing, snapshots, and metrics are user-triggered, delayed, or backgrounded.
- Full document metrics avoid synchronous recalculation per keystroke.
- Large preview uses explicit trigger and limit/truncation behavior.
- Large export and snapshot commands show progress if they take noticeable time.

Add performance probes only when a feature touches heavy paths.

